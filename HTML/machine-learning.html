Machine Learning & Decision Making

This project focused on building predictive models to understand what drives customer engagement and service purchases for a telecommunications provider. Business goals were turned into machine learning tasks, with clear targets and analysis units. The data was cleaned and prepared to improve model performance while handling bias and avoiding overfitting. Different algorithms were tested and compared using suitable metrics. Key features were identified, and insights were used to make recommendations that support better customer decisions and improve retention.

Data Preprocessing

In the first step, Target, Ordinal, and One-Hot Encoding methods are tested with a Decision Tree model using pipelines. GridSearchCV is used for tuning, and performance is evaluated with cross-validation based on MAE and R¬≤ scores to identify the most effective encoding method.

Feature Engineering & Permutation Importance

Next, permutation importance is used to identify the most influential features in the model by measuring the change in performance when each feature is randomly shuffled. Based on these insights & domain knowledge, feature engineering is applied to create new variables or transform existing ones, aiming to enhance model performance and capture hidden patterns in the data

Feature & Model selection

Feature selection is done through multiple cycles, using domain knowledge and permutation importance to refine feature sets. Models are evaluated based on performance and suitability. Random Forest is chosen over Decision Tree for its better generalization and ability to handle feature interactions, reflected in improved MAE and R¬≤ scores.

Recommendations To Stakeholders

Next, SHAP analysis is used to explain model predictions and highlight key drivers influencing the target variable. These insights are then used to make actionable recommendations for stakeholders, supporting informed decision-making.

Model Evaluation & Limitations

Finally, the model is evaluated on random samples where predictions are either accurate or significantly off. This helps uncover the model‚Äôs limitations and ensures stakeholders understand potential weaknesses to be mindful of when applying the model in real-world scenarios.
