(Each month reflects a project phase, from user requirements to deployment)

Date Description Ref No. Phase Details
01 Mar 2025 Requirement Gathering with Stakeholders REF001 Requirement

Requirement Gathering & Business Understanding

In the initial phase, we conducted calls and workshops with stakeholders to understand the business context, including department-specific operations, SLA tracking, channel performance, and key metrics that the dashboard should reflect. We clarified objectives, expectations, and the acceptance criteria for the dashboard. This included confirming the scope of metrics, required reports, and visualizations, ensuring alignment between users' needs and data availability. Sample dashboards and mockups were shared via PowerPoint to facilitate feedback and guide the design process.

01 Apr 2025 Preliminary Analysis REF002 Data Analysis

Preliminary Analysis

During this phase, we conducted preliminary data analysis in Excel to understand sample datasets and identify any inconsistencies, missing values, or formatting issues. This step allowed us to determine what data needed processing or cleansing before ingestion. Findings from Excel analysis were documented and shared with stakeholders to ensure alignment on data quality requirements. These insights later informed the SQL view logic to accurately transform and structure the data for the dashboard.

01 May 2025 Hive SQL View Developed REF003 SQL

SQL View Logic

As we progress into the second month, the focus shifts to iterative discussions with users to confirm and refine the business logic. During this stage, we validate exactly what data is required, which columns need to be included, and how they should be transformed to meet business needs. These insights are then translated into SQL logic, where we begin building the actual views that will drive the dashboard.
01 June 2025 Dashboard Visual Completed REF004 Dashboard

Finalized Dashboard (Final Phase)

In the third month, once the SQL views are finalized and signed off, we begin linking them to Power BI. Using Power Query M, we connect to the data stored in Hive or Hue and import it into the .pbix file. From there, we create the required visuals and dashboards tailored to the user's needs. The logic behind each chart or metric is implemented using a combination of DAX formulas and Query M transformations. The design and logic are then iteratively verified with the user to ensure the final dashboard meets their expectations before production deployment.
01 July 2025 User Testing Phase REF005 Testing

User Testing & Feedback

In this phase, users tested the dashboard to validate data accuracy, business logic, and visualizations. Feedback was collected iteratively to identify gaps, discrepancies, or improvements needed. Based on user input, modifications were made to data transformations, SQL views, or dashboard visuals to ensure the final product met business requirements and expectations.

01 August 2025 Deployment & Maintenance REF006 Deployment

Deployment & Maintenance

Once the dashboard and data were stabilized and signed off by users, the final artefacts were prepared for deployment into the production environment. This included finalizing SQL views, Power BI reports, and related documentation. Post-deployment, ongoing maintenance and support were planned to address any issues, update dashboards with new requirements, and ensure consistent data quality.
